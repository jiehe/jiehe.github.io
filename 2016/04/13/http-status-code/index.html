基本状态码

从事web开发，要懂得基本的http状态码的含义，有利于排查问题，定位问题源头。从而解决问题。

2叉叉 状态码代表成功

2叉叉 状态码代表请求成功。要掌握的是200、200(from cached)。

  • 200 代表成功。
  • 200(from cache) 代表静态资源之前请求过，在响应头设置了过期时间(Expires)，当再次请求的时候没有过期就会直接读取本地缓存，不与服务器发生请求，提高响应速度和节省带宽。
  • 其他 2叉叉 略。

3叉叉 状态码代表重定向

3叉叉 状态码代表重定向。要掌握的是304(Not Modified)、302。

  • 304(Not Modified)重定向也是针对静态资源请求， 代表静态资源之前请求过，在响应头设置了最后修改时间(Last-Modified)，当再次请求的时候在请求头带(If-Modified-Since: Last-Modified)最后修改时间去询问服务器静态资源有没有修改过，没有修改过就返回 304(Not Modified)让浏览器读取本地缓存，提高响应速度和节省带宽。这是http1.0的方式， http1.1也兼容。
  • http1.1 304(Not Modified)重定向的方式有所扩展， 静态资源之前请求过，在响应头设置了一个标示(Etag)，当再次请求的时候在请求头带(If-None-Match: Etag)标示去询问服务器静态资源有没有修改过，没有修改过就返回 304(Not Modified)让浏览器读取本地缓存，提高响应速度和节省带宽。
  • 302 重定向是后台重定向浏览器的链接， 当请求返回 302 状态码的时候，就要留意响应头里面的Location的链接， 该请求会被重定向到Location的链接。

4叉叉 状态码代表无权访问

4叉叉 状态码代表无权访问。要掌握的是402、403、404， 都是代表请求方无权访问。

  • 402、403 代表的是你无权访问， 常见于内网资源， 一般会返回一个页面告诉你为什么无权访问，想访问找谁。
  • 404 无权访问代表的是请求链接或页面找不到， 属于无权访问的一种。

5叉叉 服务器出错

5叉叉 状态码代表服务器出错， 常见场景是服务器程序出错，请求参数出错，无权访问导致服务器程序出错。
